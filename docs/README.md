## Documentation Standards

### For Users
- Start with context ("What is this?")
- Provide step-by-step instructions
- Include troubleshooting
- Use examples
- Keep it concise

### For Developers
- Explain architecture first
- Provide code examples
- Document common tasks
- Link to related docs
- Keep it up-to-date

### For Contributors
- Clear guidelines
- Code style examples
- Testing procedures
- PR process
- Be welcoming

## Updating Documentation

When adding features:
1. Update relevant existing docs
2. Add new docs if needed
3. Update this index
4. Add cross-references
5. Update CHANGELOG.md
